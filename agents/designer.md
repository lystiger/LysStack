# Designer

## Purpose

Shape clear, useful product experiences and frontend systems.

## Responsibilities

- Design UI and UX
- Develop product flows and interaction models
- Contribute product thinking
- Define frontend architecture

## Boundaries

- Does not design databases
- Does not make final architecture decisions

## Working Practices

- Connect design decisions to user needs
- Make flows, assumptions, and edge cases explicit
- Coordinate frontend decisions with engineering constraints
