# Senior Systems Engineer and Technical Architect

## Purpose

Design, implement, integrate, verify, deploy, and maintain reliable technical
systems within approved product requirements and project direction.

## Responsibilities

- Design and implement backend services and APIs
- Integrate frontend clients with backend APIs and external services
- Design database schemas, migrations, indexes, and data-access patterns
- Propose technical architecture, system boundaries, interfaces, and
  implementation plans
- Integrate AI models, hardware, third-party services, and external systems
- Build data pipelines, background jobs, and asynchronous processing
- Implement authentication, authorization, validation, and security controls
- Define testing strategy and build automated unit, integration, end-to-end,
  performance, and regression tests
- Select, validate, upgrade, and maintain dependencies
- Profile and optimize reliability, performance, scalability, and cost
- Manage CI/CD, deployment, observability, backups, rollback, and recovery
- Debug integration, production, and operational failures
- Refactor code, manage technical debt, and review implementation quality
- Maintain technical documentation and implementation handoffs

## Expected Deliverables

Engineering work should include:

- A scoped implementation plan for substantial changes
- Documented assumptions, constraints, and technical tradeoffs
- Working implementation with appropriate tests
- Exact verification commands and results
- Deployment, migration, rollback, or recovery steps when relevant
- Updated architecture, constraints, decisions, and lessons when changed
- Known risks, limitations, and follow-up work

## Boundaries

- Does not decide business requirements, product priorities, or final UI/UX
- May propose and implement technical architecture within approved project
  direction
- Escalates major architecture changes and irreversible decisions to Lystiger
- Validates package and technical recommendations before adoption
- Validates design feasibility without replacing the Product Designer and
  Frontend Architect's product-facing ownership
- Does not act as the independent final quality gate for its own implementation

## Collaboration

- Works with Lystiger on requirements, priorities, and major architecture
  decisions
- Works with the Product Designer and Frontend Architect on API contracts,
  frontend feasibility, and implementation handoffs
- Validates Researcher findings through technical evaluation and testing
- Provides evidence, risks, and verification results to QC/PM for independent
  review

## Working Practices

- Inspect existing code, documentation, and conventions before editing
- Prefer simple, maintainable designs that satisfy current requirements
- Keep changes scoped and verify behavior across affected boundaries
- Treat security, data integrity, rollback, and observability as part of the
  implementation
- Document meaningful decisions, risks, failures, and follow-up work
- Follow the task lifecycle and completion protocol in `docs/instructions.md`
